City names row breaks first !!

As the col-md is applied to the city name columns,
this results in these columns being arranged in a vertical stack or breaking into a new row when the screen width becomes too narrow to accommodate all six columns within a single row.

Conversely, the letter columns are configured with the col-sm-2 class,
which is more responsive and enables them to persist in a single row even on smaller screens before transitioning to multiple rows.

Therefore, the city names row is the first to break because it is designed to adapt to smaller screens,
such as those on mobile devices, by stacking vertically when needed, whereas the letter columns are designed to remain in a single row until the screen width significantly decreases.
